### Navigation Rules

**No separate navigation.md files needed** - structure is self-documenting:
- Start with topic `overview.md` (main entry point)
- Related subtopics in dedicated folders (e.g., `graph-types/`)
- Cross-references handled via inline links
- All paths relative to current file

**Finding content:**
- Main topics: Look for `overview.md` or `00-Index.md`
- Detailed guides: Look in topic-specific subfolders
- Supporting materials: Check `assets/` subdirectories

## File & Folder Naming Conventions

**ALWAYS follow these rules when creating or renaming files/folders:**

### 1. Use kebab-case (lowercase-with-hyphens)

✅ **Correct:**
- `data-visualization/`
- `overview.md`
- `dashboard-personas.md`
- `graph-types/`

❌ **Incorrect:**
- `Data-Visualization/` (uppercase)
- `01-graphs & their design.md` (spaces, special chars, numbers)
- `NAVIGATION_GUIDE.md` (all caps)
- `graphs/` (when more specific name exists)

### 2. No Special Characters

- Use hyphens, NOT spaces
- No ampersands (`&`), parentheses, or special symbols
- Exception: Numbers in data examples (e.g., `100-users-example.csv`)

### 3. Descriptive Names

- `graph-types/` NOT `charts/` or `graphs/`
- `overview.md` NOT `index.md` or `main.md`
- `navigation.md` NOT `nav.md` or `guide.md`

### 4. No Arbitrary Numbering

❌ Don't use: `01-introduction.md`, `02-basics.md`, `100-data-visualization/`
✅ Use descriptive names that indicate content purpose

## Cross-Referencing Standards

### 1. Always Use Relative Links

✅ `[Bar Chart](./graph-types/bar-chart.md)`
✅ `[Overview](../overview.md)`

❌ `/Users/username/...` (absolute paths)
❌ `file:///...` (file URLs)

### 2. Required Cross-References

**Every detailed guide must have:**

1. **Back link at top:**
   ```markdown
   **← [Back to Main Topic](../overview.md)**
   ```

2. **"See Also" section at bottom:**
   ```markdown
   ## See Also

   **Related Topics:**
   - [Related Topic 1](./related-1.md)
   - [Related Topic 2](./related-2.md)

   **Related Concepts:**
   - [Concept in Main Doc](../overview.md#concept-section)
   ```

3. **Inline links** when mentioning other chart types or concepts

### 3. Link Text Should Be Descriptive

✅ `[Bar Chart Guide](./bar-chart.md)` — Clear what you're linking to
❌ `[Click here](./bar-chart.md)` — Not descriptive

## Document Structure Standards

### Main Overview Documents

**Structure:**
1. Title and brief description
2. Table of contents (for long docs)
3. Main content in numbered parts
4. Summary/Checklist at end
5. Cross-references to detailed guides

### Detailed Guide Documents

**Required sections:**
1. Title
2. Back link to overview
3. History & Purpose (why it exists)
4. When to Use / Not to Use
5. Design Principles
6. Common Issues & Fixes
7. Examples (good and bad)
8. Quick Checklist
9. See Also section

### Persona/Audience Documents

**Required sections:**
1. Overview table (quick comparison)
2. Detailed sections per persona
3. Decision trees
4. Examples
5. Cross-references to relevant tools/charts

## Markdown Conventions

### Headers

- Use ATX-style headers (`#`, `##`, `###`)
- No trailing hashes
- One blank line before and after headers

### Lists

- Use `-` for unordered lists (not `*` or `+`)
- Use `1.` for ordered lists
- Indent nested lists with 2 spaces

### Code Blocks

- Use fenced code blocks with language identifier:
  ````markdown
  ```python
  code here
  ```
  ````

### Tables

- Use GitHub-flavored markdown tables
- Align headers with `|---|` separator
- Keep columns aligned for readability

### Emphasis

- **Bold** for key terms: `**term**`
- *Italic* for emphasis: `*emphasis*`
- `Code` for technical terms: `` `code` ``

## Adding New Content

**When adding new topics, always:**

1. ✅ Create folder with kebab-case name
2. ✅ Create `overview.md` or main topic file (NOT README.md)
3. ✅ Create `assets/` with `images/`, `examples/`, `templates/` subfolders
4. ✅ Add cross-references to existing content
5. ✅ Update parent folder's index if it exists
6. ✅ Follow the document structure standards above

**Do NOT create:**
- ❌ `README.md` in topic folders (use `overview.md` instead)
- ❌ `navigation.md` files (structure is self-documenting)
- ❌ `CONTRIBUTING.md` or other meta files per topic

**Example workflow:**

```bash
# Creating new topic: "metric-design"
mkdir metric-design
cd metric-design
touch overview.md
mkdir -p assets/{images,examples,templates}
mkdir metric-types
# Then create content following standards
```
